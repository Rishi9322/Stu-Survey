PROJECT OVERVIEW
================================================================================
You are building a Student Feedback Management System in React (TypeScript) with 
Firebase (Firestore + Authentication) as the backend. The application is a 
comprehensive educational feedback platform that allows students, teachers, and 
admins to manage surveys, collect feedback, and handle suggestions/complaints 
with AI-powered analytics.


CORE DATABASE COLLECTIONS (FIRESTORE)
================================================================================

Create the following Firestore collections with these document structures:

1. USERS COLLECTION
   - id (string, uid from Firebase Auth)
   - name (string)
   - email (string)
   - password (hashed if needed, else use Firebase Auth)
   - role (enum: 'student' | 'teacher' | 'admin')
   - isActive (boolean)
   - createdAt (timestamp)
   - updatedAt (timestamp)

2. SURVEYS COLLECTION
   - id (string, auto-generated)
   - title (string)
   - description (string)
   - isActive (boolean)
   - createdAt (timestamp)
   - updatedAt (timestamp)

3. QUESTIONS COLLECTION
   - id (string, auto-generated)
   - surveyId (string, reference to surveys)
   - question (string)
   - questionType (enum: 'rating' | 'text' | 'multiple_choice')
   - isActive (boolean)
   - order (number)
   - createdAt (timestamp)
   - updatedAt (timestamp)

4. RESPONSES COLLECTION
   - id (string, auto-generated)
   - surveyId (string, reference)
   - studentId (string, reference)
   - teacherId (string, reference)
   - questionId (string, reference)
   - rating (number, 1-5, optional)
   - textResponse (string, optional)
   - submittedAt (timestamp, optional)
   - createdAt (timestamp)
   - updatedAt (timestamp)

5. SUGGESTIONS_COMPLAINTS COLLECTION
   - id (string, auto-generated)
   - subject (string)
   - description (string)
   - type (enum: 'complaint' | 'suggestion')
   - submittedByRole (enum: 'student' | 'teacher' | 'admin')
   - status (enum: 'open' | 'pending_review' | 'resolved' | 'closed')
   - resolutionNotes (string, optional)
   - resolvedBy (string, reference to users, optional)
   - resolvedAt (timestamp, optional)
   - createdAt (timestamp)
   - updatedAt (timestamp)


CORE FEATURES & MODULES
================================================================================

1. AUTHENTICATION MODULE
   - Firebase Auth integration (email/password, optional: Google/social login)
   - Login page with email validation
   - Registration page with role selection (student/teacher, admin by invitation only)
   - Password reset functionality
   - Session management and token refresh
   - Protected routes based on user role

2. USER MANAGEMENT (ADMIN)
   - View all users with filters (by role, status, email)
   - Create, update, delete users
   - Bulk user import (CSV/Excel with validation)
   - Bulk user export (CSV/Excel format)
   - User import error reporting
   - Set user roles (student, teacher, admin)
   - Activate/deactivate users

3. SURVEY MANAGEMENT (ADMIN & TEACHER)
   - Create surveys with title and description
   - Add questions to surveys (rating, text, multiple choice)
   - Reorder questions
   - Preview surveys
   - Activate/deactivate surveys
   - Edit existing surveys
   - Delete surveys
   - View survey statistics and responses

4. STUDENT SURVEY TAKING
   - List available active surveys
   - Display surveys with all question types
   - Save responses (rating 1-5, text input, multiple choice selection)
   - Submit survey responses with timestamp
   - View survey progress
   - Prevent duplicate submissions for same user/survey
   - Display confirmation on successful submission

5. TEACHER FEEDBACK DASHBOARD
   - View surveys created by them
   - View all responses to their surveys
   - View student ratings and feedback
   - Filter responses by student, date range, rating
   - Export responses to CSV/Excel

6. ADMIN ANALYTICS DASHBOARD
   - Overall statistics (total users, surveys, responses, complaints)
   - Survey completion rates
   - Response trends over time
   - Complaint/suggestion statistics
   - User activity metrics
   - Data visualization (charts and graphs)

7. SUGGESTIONS & COMPLAINTS MODULE
   - Submit complaints/suggestions (anonymous option available)
   - View all submissions (admin only)
   - Filter by type (complaint/suggestion), status, date range
   - Change status (open → pending_review → resolved/closed)
   - Add resolution notes
   - Track which admin resolved it
   - Auto-timestamp resolutions

8. AI INTEGRATION (Optional but Present)
   - Sentiment analysis for text responses
   - Topic extraction from suggestions/complaints
   - Automated response categorization
   - Predictive analytics for survey trends
   - API integration with external AI services (Grok, OpenRouter, local Python)
   - Real-time data analysis

9. DATA IMPORT/EXPORT
   - CSV/Excel user import with validation and error handling
   - Batch user creation from imported data
   - Email notifications to imported users
   - User export with filters
   - Response data export
   - Survey templates export/import

10. REPORTING & DOCUMENTATION
    - Generate PDF reports for surveys
    - HTML-based analytics reports
    - Documentation pages (about, features, help, terms, privacy)
    - Contact form for support
